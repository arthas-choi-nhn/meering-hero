# PRD: 사내 회의록 자동화 서비스 (Internal Meeting Transcription Service)

## 1. 개요

### 1.1 배경

팀 내부 회의에는 프로젝트 기밀, 인프라 구성, 보안 정책 등 민감한 내용이 포함된다. Tiro와 같은 퍼블릭 STT/요약 서비스를 사용할 경우 음성 데이터가 외부 서버로 전송되어 보안 정책에 위배된다.

이를 해결하기 위해 **음성 인식(STT)을 사용자의 MacBook에서 로컬로 처리**하고, 요약 및 최종 결과물은 **Dooray Wiki에 저장**하는 사내 전용 회의록 자동화 서비스를 구축한다.

### 1.2 목표

- 회의 음성을 **실시간으로 텍스트 변환**(STT)하되, 음성 데이터가 사내 네트워크 외부로 나가지 않도록 한다.
- 회의 종료 후 **LLM 기반 자동 요약**을 제공한다.
- 전사 내용과 요약을 **Dooray Wiki에 내보내기**하여 팀 내 공유 및 아카이빙한다.
- 사용자의 MacBook Pro(Apple Silicon) 리소스를 활용하여 **별도 GPU 서버 없이** 운영한다.

### 1.3 대상 사용자

- NHN Cloud 인프라/개발팀 구성원
- MacBook Pro(Apple Silicon M1/M2/M3/M4) 사용자
- Claude Code가 설치 및 로그인된 환경

### 2.3 LLM 요약

| 요구사항 | 설명 | 우선순위 |
|---------|------|---------|
| Claude Code CLI 연동 | `claude -p` 명령으로 구독 쿼터 기반 요약 (추가 비용 없음) | P0 |
| 템플릿 기반 요약 | 회의록, 데일리 스크럼, 기술 리뷰, 액션아이템 등 미리 정의된 템플릿 | P0 |
| 사용자 정의 템플릿 | 사용자가 직접 요약 형식을 정의할 수 있음 | P2 |
| 로컬 LLM 폴백 | Ollama(EXAONE, Qwen 등) 기반 완전 오프라인 요약 옵션 | P2 |

**Claude Code CLI 제약사항:**

- OAuth 토큰(`sk-ant-oat01-*`)은 Anthropic Messages API에서 사용 **불가**
- CLI subprocess 방식(`claude -p`)으로 호출하며, 요청당 **3~5초 오버헤드** 발생
- Pro/Max 구독 쿼터에서 차감되며, 쿼터 소진 시 사용 불가

## 3. 비기능 요구사항

### 3.1 보안

- **음성 데이터**: STT가 로컬에서 처리되므로 음성 데이터가 외부로 전송되지 않는다.
- **전사 텍스트**: LLM 요약 시 Claude API를 통해 전사 텍스트가 Anthropic으로 전송될 수 있다. 극비 회의의 경우 로컬 LLM(Ollama) 사용을 권장한다.
- **Dooray 인증**: Dooray API 토큰을 macOS Keychain에 안전하게 저장한다.
- **로컬 저장소**: SQLite 데이터는 `~/Library/Application Support/` 하위에 저장하며, macOS 파일시스템 암호화(FileVault)에 의존한다.

### 3.2 성능

- **STT 지연**: 실시간 전사 시 발화 종료 후 **2초 이내** 텍스트 표시
- **요약 응답**: Claude Code CLI 호출 후 **15초 이내** 결과 반환 (1시간 회의 기준)
- **리소스 점유**: STT 실행 중 MacBook 전체 CPU 사용률 **50% 이하** 유지
- **모델 로드**: 앱 시작 시 STT 모델 로드 **5초 이내**

### 3.3 호환성

- **macOS**: 13.0 (Ventura) 이상 (ScreenCaptureKit 필요)
- **Apple Silicon**: M1, M2, M3, M4 시리즈 전체 지원
- **Claude Code**: 최신 버전 설치 및 로그인 상태 필요

## 4. 사용자 플로우

```
[앱 실행 (메뉴바 상주)]
        │
        ▼
[새 회의 시작]
  ├─ 제목 입력
  ├─ 참석자 입력 (선택)
  ├─ 오디오 소스 선택 (마이크 / 시스템 / 혼합)
  └─ 맥락 힌트 입력 (선택, 전문용어 등)
        │
        ▼
[녹음 중 — 실시간 전사]
  ├─ 화면에 실시간 텍스트 표시
  ├─ partial(진행중) → final(확정) 교체
  └─ 일시정지 / 재개 가능
        │
        ▼
[녹음 종료]
        │
        ▼
[전사 결과 확인 및 편집]
        │
        ▼
[요약 생성]
  ├─ 템플릿 선택 (회의록 / 스크럼 / 기술리뷰 / 액션아이템)
  ├─ "Claude로 요약" 클릭
  ├─ Claude Code CLI 실행 → 결과 반환 (~5-15초)
  └─ 요약 결과 편집 가능
        │
        ▼
[Dooray Wiki 내보내기]
  ├─ 프로젝트 선택
  ├─ 상위 Wiki 페이지 선택
  ├─ 내보내기 실행
  └─ 생성된 Wiki 페이지 URL 표시
```

## 6. MVP 범위

**Phase 1 (MVP)**
- 마이크 입력 캡처 + 로컬 STT(whisper.cpp + Core ML)
- 실시간 전사 화면
- Claude Code CLI 기반 요약 (회의록 템플릿)
- Dooray Wiki 단일 페이지 내보내기
- SQLite 로컬 세션 저장

**Phase 2**
- 시스템 오디오 캡처 (ScreenCaptureKit)
- 화자 분리 (pyannote-audio)
- 추가 템플릿 (스크럼, 기술리뷰, 액션아이템)
- 세션 검색 및 과거 기록 열람

**Phase 3**
- 로컬 LLM 폴백 (Ollama)
- 사용자 정의 템플릿
- 사내 용어 사전 / fine-tuning
- 타임스탬프 기반 오디오 점프
